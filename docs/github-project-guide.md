# GitHub Project Guide

Day la huong dan nhanh de tao Project board cho repo `Retail_Sales`.

## 1. Tao Project

1. Mo repo tren GitHub: `https://github.com/hnguyen76/Retail_Sales`
2. Chon tab **Projects**.
3. Chon **New project**.
4. Chon template **Board**.
5. Dat ten: `Retail Sales Dashboard Roadmap`.

## 2. Tao Cot

Dung cac cot sau:

| Column | Meaning |
| --- | --- |
| Todo | Viec chua bat dau |
| In Progress | Viec dang lam |
| Review | Viec da xong, can xem lai |
| Done | Viec da hoan thanh |

## 3. Tao Issue Tu Template

1. Vao tab **Issues**.
2. Chon **New issue**.
3. Chon template **Project task**.
4. Copy mot task tu `PROJECT_PLAN.md`.
5. Submit issue.
6. Add issue vao Project board.

## 4. Task Mau Nen Tao

Tao cac issue nay truoc:

| Issue | Priority | Area |
| --- | --- | --- |
| Enable GitHub Pages deployment | High | Deployment |
| Add dashboard screenshot to README | High | Documentation |
| Add Vietnamese executive summary | Medium | Documentation |
| Add forecast section | Medium | Dashboard |
| Add customer segment deep dive | Medium | Analytics |
| Add data quality checks | Low | Data |

## 5. Workflow Mau

Khi lam viec:

1. Keo issue tu `Todo` sang `In Progress`.
2. Sua code hoac tai lieu.
3. Commit va push.
4. Kiem tra lai dashboard/report.
5. Keo issue sang `Review`.
6. Neu on, keo sang `Done`.

Day la cach bien repo thanh mot project co quy trinh ro rang, rat tot cho
portfolio va phong van.
